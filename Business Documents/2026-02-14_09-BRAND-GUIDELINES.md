# Sassy Browser — Brand Guidelines

**Sassy Consulting LLC | February 2026**

## Brand Identity

### Brand Name
- **Full name:** Sassy Browser
- **Company name:** Sassy Consulting LLC
- **Abbreviation:** SaS (internal only, not for marketing)
- **Never:** "Sassy," "The Sassy Browser," "SassyBrowser" (one word)

### Tagline
**Primary:** *"Your browser. Your data. Always."*

**Secondary options:**
- "Privacy without compromise."
- "Built for you, not advertisers."
- "The browser that works for you."

### Brand Voice

| Attribute | Description | Example |
|-----------|-------------|---------|
| **Confident** | We know what we built is good | "77,000 lines of privacy-first code." |
| **Direct** | No corporate speak, no jargon | "We don't track you. Period." |
| **Technical** | We respect our users' intelligence | "ChaCha20-Poly1305 encryption, not marketing buzzwords." |
| **Approachable** | Not intimidating, not condescending | "Privacy should be the default, not a setting you have to find." |
| **Principled** | Our values come through | "Zero telemetry isn't a feature — it's a promise." |

### What We Are NOT
- Not snarky or sarcastic (despite the name)
- Not aggressive toward competitors (state facts, not attacks)
- Not corporate or bureaucratic
- Not preachy about privacy (show, don't lecture)

### Logo Guidelines

#### Logo Variations
1. **Full logo** — Icon + "Sassy Browser" text
2. **Icon only** — For small spaces, favicons, app icons
3. **Text only** — For document headers, legal contexts

#### Logo Usage Rules
- Minimum clear space: 1x icon height on all sides
- Minimum size: 24px height for icon, 120px for full logo
- Never stretch, rotate, or add effects
- Never place on busy backgrounds without contrast backing
- Approved background colors: navy, white, black

#### Logo Don'ts
- Don't change the colors
- Don't add drop shadows or gradients
- Don't put text over the logo
- Don't use low-resolution versions
- Don't combine with other logos

## Messaging Guidelines

### Key Messages by Audience

#### For Privacy Enthusiasts
- Lead with: zero telemetry architecture
- Emphasize: no Google dependencies, all data stays local
- Proof points: 77K LOC in Rust, open for audit

#### For Parents
- Lead with: built-in family safety
- Emphasize: no additional software needed, COPPA compliant
- Proof points: age-appropriate profiles, NSFW detection

#### For Developers
- Lead with: MCP integration, built-in tools
- Emphasize: replaces $5K/year in software
- Proof points: 456 tests, extension API, dev tools

#### For Enterprise
- Lead with: compliance-ready, zero cloud dependency
- Emphasize: on-device data processing, admin controls
- Proof points: encryption specs, sandbox architecture

### Boilerplate (Company Description)

**Short (25 words):**
Sassy Browser is a privacy-first web browser built in Rust with zero telemetry, built-in security tools, and family safety features.

**Medium (50 words):**
Sassy Browser is a privacy-first web browser built from the ground up in Rust. With 77,000 lines of code, zero Google dependencies, and a zero-telemetry architecture, it provides advanced anti-tracking, encrypted password management, family safety profiles, and developer tools — all without sending a single byte of user data to any server.

**Long (100 words):**
Sassy Browser is a next-generation web browser built entirely in Rust by Sassy Consulting LLC. Designed from the ground up with privacy as a core architectural principle — not a bolted-on feature — it processes all user data exclusively on-device. The browser features a 4-layer sandbox system, active fingerprint poisoning, behavioral mimicry to prevent tracking, ChaCha20-Poly1305 encrypted password vault, built-in family profiles with age-appropriate controls, and a self-healing watchdog system. With 77,000 lines of code, 456 unit tests, and zero Google-owned dependencies, Sassy Browser represents a fundamental rethinking of what a browser should be: a tool that works for users, not advertisers.

## Content Guidelines

### Blog Posts
- Title: Clear, benefit-driven, under 60 characters
- Length: 800–1,500 words
- Tone: Technical but accessible
- Always include: what problem this solves for users
- End with: clear CTA (download, subscribe, share)

### Social Media
- **Twitter/X:** Technical facts, privacy tips, development updates
- **Reddit:** Detailed technical posts, community engagement
- **YouTube:** Demo videos, architecture explainers, privacy education
- **LinkedIn:** Company milestones, hiring, industry analysis

### Documentation
- Clear, concise, example-heavy
- Code examples in Rust with comments
- Version-tagged (match browser release versions)
- Accessible language (avoid jargon where possible)

## Legal Usage

### Required Disclaimers
- Website footer: "Sassy Browser is a product of Sassy Consulting LLC"
- Privacy claims: link to privacy policy
- Security claims: "based on architectural design" (until audited)

### Trademark Usage
- Always capitalize: "Sassy Browser"
- First mention in any document: Sassy Browser™ (until registered, then ®)
- Never use as a verb: "Sassy your browsing" ← NO
